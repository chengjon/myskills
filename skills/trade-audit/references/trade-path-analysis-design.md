## 分析框架：3层×5维度

```
L1 单笔(V3审计已有) → L2 交易路径(Trade Path) → L3 行为画像(Behavior Profile)
```

5维度：入场质量 / 持仓管理 / 风控执行 / 仓位集中度 / 行为模式

## 路径切分算法

路径 = 同一股票从首次买入到完全沽清的所有交易序列。同一股票可有多条路径。

切分条件（满足任一即切分）：
1. 日内交易(buy_date==sell_date) → 单独成路径(is_intraday=True)
2. 空仓超过自适应阈值天（非硬编码5天）
3. 市场趋势状态变化(bull→bear / bear→bull)
4. gap > 该股平均间隔×3（自适应突变检测）

**专家审核意见**：gap_days=5硬编码无统计依据。实测gap分布均匀(0-1天54笔,2-3天42,6-10天40,11-20天49)无自然断裂点，需自适应方法。

## 路径分类(7种)

intraday / hunter / swing / sticky / add_down / late_stop / missed

## 6种审核盲区及对策

| 盲区 | 对策 | 优先级 |
|------|------|--------|
| gap硬编码 | gap分布统计化+趋势条件切分 | P0 |
| 无风险调整收益 | 加Sharpe/Pain Index/Calmar/MaxDD Duration | P1 |
| 小样本无显著性 | Wilson CI + 最低30笔门槛 + Cohen's d效应量 | P0 |
| 缺组合视角 | 同时活跃路径数vs收益/路径重叠/机会成本 | P2 |
| 交易成本缺失 | 佣金+印花税+滑点+成本占比+扣成本净盈亏 | P1 |
| 行为无时间维度 | 按年度切片+行为改善指数+关键事件标记 | P2 |

## 交叉分析设计(4项)

1. BOLL位置×持仓天数 → 胜率矩阵
2. 冲动交易×sell_verdict → 冲动卖出后果
3. 连亏次数×下次交易结果 → 验证冷却规则
4. 交易顺序效应 → 第1笔vs第N笔

每组附 Wilson 95% CI + 样本量标注(≥30笔) + Cohen's d效应量
